# Публикация сайта ГАЗОБЛОК34 на Vercel

Сайт статический (HTML + JS), поэтому разворачивается на Vercel за пару минут и
работает бесплатно. Ниже — два способа: совсем простой (через сайт Vercel) и
через GitHub (удобно для дальнейших обновлений).

---

## Что уже готово в проекте
- `index.html` — главная страница (Vercel открывает её по адресу `/`).
- `Газоблок34.html` — оставлен как авто-переадресация на `index.html`.
- `admin.html` — панель управления (под паролем).
- `vercel.json` — настройки кэша и заголовков (трогать не нужно).
- `robots.txt`, `sitemap.xml` — для поисковиков.
- `app/`, `assets/` — стили, скрипты, изображения.

Папку `telegram-bot/` выкладывать на сайт **не нужно** — бот работает отдельно.

---

## Способ 1. Загрузка папки (самый простой, без программ)

1. Зайдите на **vercel.com** → зарегистрируйтесь (можно через почту или GitHub).
2. Скачайте проект к себе на компьютер (кнопкой «Скачать» в чате) и распакуйте.
3. В Vercel нажмите **Add New… → Project → Deploy** (или раздел **Deployments**).
   В простом режиме Vercel разрешает перетащить папку прямо в браузер
   (**drag-and-drop**). Перетащите распакованную папку проекта целиком.
4. Нажмите **Deploy**. Через минуту получите адрес вида
   `https://gazoblok34.vercel.app` — сайт уже в интернете.

> Перед загрузкой можно удалить папку `telegram-bot` из распакованной папки —
> она к сайту не относится.

---

## Способ 2. Через GitHub (удобно обновлять)

1. Создайте репозиторий на **github.com** и загрузите туда файлы проекта
   (кроме папки `telegram-bot`).
2. На **vercel.com** → **Add New… → Project** → выберите этот репозиторий.
3. Framework Preset — **Other** (это статический сайт, сборка не нужна).
   Output/Root каталог — корень репозитория. Нажмите **Deploy**.
4. Теперь любое изменение в GitHub автоматически обновляет сайт.

---

## Свой домен gazoblok34.ru

1. В проекте Vercel → **Settings → Domains → Add**.
2. Введите `gazoblok34.ru` и `www.gazoblok34.ru`.
3. Vercel покажет, какие **DNS-записи** прописать у регистратора домена (r01.ru):
   - **A** запись, имя `@`, значение `216.198.79.1`
   - **CNAME**, имя `www`, значение из Vercel (вида `…vercel-dns-017.com.`)
4. После обновления DNS (от 10 минут до пары часов) сайт откроется на вашем домене,
   Vercel сам выпустит бесплатный SSL (https).

## После публикации — обязательно

1. **Яндекс.Вебмастер** (webmaster.yandex.ru): добавьте сайт, подтвердите права,
   отправьте `sitemap.xml` — чтобы сайт попал в поиск Яндекса.
2. **Google Search Console** (по желанию) — то же самое для Google.
3. **Яндекс.Метрика**: создайте счётчик и впишите его **номер** в админке
   (вкладка «Аналитика») — пойдёт статистика посещений.
4. **Telegram-бот**: запустите `telegram-bot/bot.py` на сервере/компьютере,
   чтобы заявки из бота приходили вам (см. `telegram-bot/README.md`).

---

## Важно знать
- Контент, который вы меняете в **админке**, сохраняется **в браузере того
  устройства**, где вы редактируете. Чтобы правки видели все посетители на своих
  устройствах, изменения нужно вносить в файл `app/data.js` и заново публиковать
  (перетащить обновлённую папку в Vercel или закоммитить в GitHub). Админка удобна
  для подготовки и просмотра; «постоянные» правки для всех — через повторную
  публикацию. Если нужна полноценная админка с общей базой — это уже отдельный
  проект с сервером, могу помочь отдельно.
